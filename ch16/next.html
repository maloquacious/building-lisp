Where do we go from here?

The goal of this project was to demonstrate an easy implementation of LISP. There is not much point in optimizing or mindlessly implementing library functions — this work has already been done in other projects.

Take a look at some other implementations of Building LISP:

  • kimtg/ToyList
  • jpaquim/building-lisp-zig

Here are some possible extensions which might prove interesting:

  • Expose continuations with call/cc
  • Other numeric types (float, rational, bignum, complex)
  • String, vector and boolean types
  • I/O support
  • Calls to system libraries
  • JIT (integrate with LLVM, for example)
  • Slab allocation
  • Alternative GC

Now it's time to stop messing about in Go and build something in LISP instead!

That's all, folks.
